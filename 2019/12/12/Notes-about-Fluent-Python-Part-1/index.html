Notes about Fluent Python Part 1

Python 官方文档早已扫过一遍，断断续续也写了两年多的 Python 代码了，但还是觉得缺了什么。没有整体感，这是出现在我头脑里的第一个想法。或许是到了需要进阶一下的时候，所以有了这些读书笔记 —— 关于 Fluent Python。以下是第一部分。

Chapter 1 The Python Data Model

Object are Python’s abstraction for data. All data in a Python program is represented by objects or by relations between objects. In a sense, and in conformance to Von Neumann’s model of a “Stored Program Computer”, Code is also represented by objects.

我一开始理解这部分的时候，老是去想它的具体实现方式（可能是受 C++ 学习经验的影响）。但其实它这里想表述的是 Python 所构建的那个体系，它是如何去抽象现实世界里的事物的，注意它上面说的是 abstraction，与抽象相对应的是具体，也就是实现（下一层的实现可以是 CPython，JPython等）。但这里只说抽象，也就是只谈 Python 体系的结构。理解了这一点，一切都顺理成章。

Python Data Model 中最亮眼的就是：special method，让它变得极易扩展？（这里会补充对比例子，同一种功能实现，对比 Python 和 C++，Java）。

Chapter 2 An Array of Sequences

Python中的 Sequence 有两种常见的分类方式：
mutable sequence vs immutable sequence
container sequence(存引用） vs flat sequence（存值）

mutable sequence 如 list, array.array, bytearray
immutable sequence 如 tuple、str、bytes

可变和不可变 指的是 它所指的那个 Python 对象是不变的，即 id(obj) 不变，但它的值你是无法保证不改变的。如 t = （[1],[2]), t 是 tuple，但 t 的内容明显是可变的，t[0].append(3)是可行的。
